Cross-Model Hooks

Delimit hooks provide governance that works across all AI coding assistants.

What Are Hooks?

Delimit hooks are governance checkpoints that trigger when code changes are made, regardless of which AI assistant or IDE is being used. They ensure consistent API governance across your entire team.

Pre-Commit Hook

The pre-commit hook runs Delimit checks before every commit:

# Install the hook
delimit activate

# Or manually add to .git/hooks/pre-commit
npx delimit-cli pre-commit-check

What the Hook Checks

  • Secret detection (API keys, passwords, tokens in staged files)
  • OpenAPI spec validation (if specs are staged)
  • Policy compliance (if a policy file exists)
  • Evidence recording (audit trail in ~/.delimit/evidence/)

PATH Shim Integration

When activated, Delimit adds a shim layer to your PATH that provides continuous governance:

delimit activate
# Adds ~/.delimit/shims to PATH via shell profile

Conditional Hooks (Claude Code)

Claude Code supports an if field on hooks that lets Delimit fire governance checks only when they matter. This keeps hooks invisible during normal coding and active only when API specs are touched.

PreToolUse: Lint only on spec edits

This hook fires only when an Edit or Write targets an OpenAPI or Swagger file:

{
  "hooks": {
    "PreToolUse": [
      {
        "type": "command",
        "command": "npx delimit-cli hook pre-tool $TOOL_NAME",
        "matcher": "Edit|Write",
        "if": "Edit && (path_matches('**/openapi*') || path_matches('**/swagger*') || path_matches('**/*.yaml') || path_matches('**/*.yml'))"
      }
    ]
  }
}

PreToolUse: Governance on commit/push

This hook fires only when a Bash command contains git commit or git push:

{
  "hooks": {
    "PreToolUse": [
      {
        "type": "command",
        "command": "npx delimit-cli hook pre-commit",
        "matcher": "Bash",
        "if": "Bash && (input_contains('git commit') || input_contains('git push'))"
      }
    ]
  }
}

SessionStart: No condition needed

The session-start hook always fires to show governance status and ledger context:

{
  "hooks": {
    "SessionStart": [
      {
        "type": "command",
        "command": "npx delimit-cli hook session-start"
      }
    ]
  }
}

Automatic installation

Running delimit setup or delimit activate installs these conditional hooks automatically. Existing hooks without if conditions are upgraded in place.

Cross-Model Compatibility

Hooks work the same way regardless of whether changes are made by:

  • Claude Code (via MCP or direct edits)
  • OpenAI Codex CLI
  • Google Gemini CLI
  • Manual editing in VS Code, Vim, etc.

The governance layer operates at the git level, not the editor level, ensuring universal coverage.

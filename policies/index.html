Policies

Control what API changes are allowed, warned, or blocked with policy presets and custom YAML configuration.

Policy Presets

Delimit ships with three built-in presets:

Strict

All breaking changes are errors. Zero tolerance for API-breaking modifications.

delimit init --preset strict

Default

Balanced approach. Critical changes (endpoint removal, field removal) are errors. Other breaking changes are warnings.

delimit init

Relaxed

All breaking changes are warnings. Nothing blocks CI. Good for early-stage APIs.

delimit init --preset relaxed

Custom Policy YAML

Create .delimit/policies.yml for fine-grained control:

# .delimit/policies.yml
rules:
  no_endpoint_removal:
    severity: error
    message: "Endpoints must not be removed without deprecation"

  no_field_removal:
    severity: error
    message: "Response fields must not be removed"

  no_required_param_addition:
    severity: error
    message: "New required parameters break existing consumers"

  warn_type_change:
    severity: warning
    message: "Type changes should be reviewed carefully"

  no_enum_removal:
    severity: warning
    message: "Removing enum values may break consumers"

Severity Levels

  • error — Blocks CI in enforce mode. Shown as red in PR comments.
  • warning — Shown in PR comments but does not block CI.
  • info — Logged for awareness only.

Using with GitHub Action

- uses: delimit-ai/delimit-action@v1
  with:
    spec: openapi.yaml
    mode: enforce
    policy_file: .delimit/policies.yml

Using with CLI

delimit lint --old v1.yaml --new v2.yaml --policy .delimit/policies.yml

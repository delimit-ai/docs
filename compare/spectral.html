Delimit vs Spectral

Spectral validates spec quality. Delimit validates spec compatibility. They solve different problems and work well together.

What Spectral Does

Spectral is an open-source linter from Stoplight. It analyzes a single OpenAPI spec and checks it against style rules: naming conventions, required descriptions, consistent casing, response structure patterns. It answers the question "Is this spec well-written?"

Spectral does not compare two versions of a spec. It has no concept of "old" vs "new." It cannot tell you what changed between releases, whether a change is breaking, or what semver bump is needed.

What Delimit Does

Delimit compares two versions of an OpenAPI spec and detects what changed. It identifies 27 change types (17 breaking, 10 non-breaking), classifies each by severity, recommends a semver bump, and generates migration guides for consumers. It answers the question "Will this change break existing API clients?"

Complementary, Not Competing

These tools address different stages of the API lifecycle:

  • Spectral — Run during spec authoring to enforce style and structure consistency.
  • Delimit — Run in CI on pull requests to catch breaking changes before merge.

Many teams run both. Spectral ensures every spec follows naming conventions. Delimit ensures no PR introduces an unintentional breaking change.

When to Use Both

If your team already uses Spectral, adding Delimit takes one line in your CI workflow. Spectral catches style violations on every commit. Delimit catches compatibility violations on every PR that touches your API spec.

# .github/workflows/api-governance.yml
- uses: stoplightio/spectral-action@v0.8.1    # Lint spec quality
- uses: delimit-ai/delimit-action@v1           # Detect breaking changes

Get Started

npx delimit-cli setup

Sets up Delimit in your repo in under a minute. Works alongside Spectral with no configuration overlap.

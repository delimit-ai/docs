Delimit vs OasDiff

Both tools diff OpenAPI specs. OasDiff gives you a binary pass/fail. Delimit gives you policy control, semver classification, and actionable PR feedback.

What OasDiff Does

OasDiff is an excellent open-source tool for detecting breaking changes in OpenAPI specs. It is written in Go, distributed as a Docker image, and provides a binary pass/fail result in CI. It has a solid breaking change catalog and an active maintainer community.

What Delimit Adds

Delimit covers the same breaking change detection but layers on governance features that teams need as they scale:

  • Policy presets — Choose strict (all errors), default (balanced), or relaxed (warnings only) without writing config.
  • Per-path scoping — Apply different policies to different API paths. Internal endpoints can be relaxed while public endpoints stay strict.
  • Semver classification — Every diff result includes a deterministic MAJOR / MINOR / PATCH / NONE recommendation.
  • Migration guide generation — Delimit generates consumer-facing migration notes explaining what changed and how to adapt.
  • PR comments with severity badges — The GitHub Action posts structured comments with breaking/non-breaking badges directly on the pull request.
  • MCP integration — Use Delimit inside Claude Code, Codex, and Gemini CLI via the MCP server.

The Ignore File Problem

OasDiff uses an ignore file to suppress known intentional changes. As your API evolves, this file grows. Every intentional breaking change requires a manual entry. Over time, the ignore file becomes a maintenance burden and a source of drift between what the file says and what the team intended.

Delimit handles this with policy rules instead. You define which change types are errors, warnings, or ignored at the policy level. Intentional changes are handled by preset selection or per-path overrides, not by maintaining a growing exception list.

Real-World Example

Projects like cal.com use OasDiff to gate breaking changes in CI. Delimit provides the same gating with less configuration overhead. Switching from OasDiff to Delimit means replacing your Docker step with a single Action line and removing the ignore file.

Get Started

npx delimit-cli setup

One command. No Docker. No ignore file. Policy presets handle the rest.

Quick Start

Get API governance running in under 5 minutes.

Option 1: CLI (Local Development)

# Install
npx delimit-cli setup

# Initialize in your project
delimit init

# Compare two specs
delimit lint --old api/v1.yaml --new api/v2.yaml

# Or diff without policy enforcement
delimit diff --old api/v1.yaml --new api/v2.yaml

Option 2: GitHub Action (CI/CD)

Add this to .github/workflows/api-governance.yml:

name: API Governance
on:
  pull_request:
    paths: ["openapi.yaml", "api/**"]

jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: delimit-ai/delimit-action@v1
        with:
          spec: openapi.yaml
          mode: advisory

That's it. Delimit will automatically compare your OpenAPI spec against the base branch and comment on the PR with any breaking changes.

Option 3: MCP Server (AI Assistants)

# Add to your MCP config
npx delimit-cli mcp

See the MCP setup guide for Claude Code, Codex, and Gemini CLI configuration.

What Happens Next

  1. Delimit diffs your old and new OpenAPI specs.
  2. It detects up to 27 types of changes (17 breaking, 10 non-breaking).
  3. It applies your policy (strict, default, or relaxed).
  4. In CI, it posts a PR comment with a detailed report.
  5. In enforce mode, it blocks the merge if breaking changes are found.

API Versioning in OpenAPI

URL path, header, and query parameter versioning strategies. Each approach has trade-offs for discoverability, caching, and maintenance. Delimit enforces version policies with path-scoped rules.

1. URL Path Versioning

The most common approach. The version is part of the URL path. Simple, visible, and cache-friendly.

# OpenAPI 3.0 — URL Path Versioning
openapi: 3.0.3
info:
  title: Orders API
  version: 2.0.0
servers:
  - url: https://api.example.com/v2
paths:
  /orders:
    get:
      summary: List orders
      operationId: listOrdersV2
      responses:
        '200':
          description: List of orders
          content:
            application/json:
              schema:
                type: object
                properties:
                  data:
                    type: array
                    items:
                      $ref: '#/components/schemas/Order'
  /orders/{id}:
    get:
      summary: Get order by ID
      operationId: getOrderV2
      parameters:
        - name: id
          in: path
          required: true
          schema:
            type: string
            format: uuid
      responses:
        '200':
          description: Order details
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/Order'

2. Header Versioning

The version is specified via a custom request header. Keeps URLs clean but is less discoverable and harder to test in a browser.

# OpenAPI 3.0 — Header Versioning
openapi: 3.0.3
info:
  title: Orders API
  version: 2.0.0
servers:
  - url: https://api.example.com
paths:
  /orders:
    get:
      summary: List orders
      operationId: listOrders
      parameters:
        - name: X-API-Version
          in: header
          required: false
          schema:
            type: string
            enum: ['1', '2']
            default: '2'
          description: API version. Defaults to latest.
        - name: Accept
          in: header
          required: false
          schema:
            type: string
            default: application/vnd.example.v2+json
          description: Content negotiation with version (alternative)
      responses:
        '200':
          description: List of orders
          content:
            application/json:
              schema:
                type: object
                properties:
                  data:
                    type: array
                    items:
                      $ref: '#/components/schemas/Order'

3. Query Parameter Versioning

The version is a query parameter. Easy to use and test, but pollutes the query string and can conflict with caching.

# OpenAPI 3.0 — Query Parameter Versioning
openapi: 3.0.3
info:
  title: Orders API
  version: 2.0.0
servers:
  - url: https://api.example.com
paths:
  /orders:
    get:
      summary: List orders
      operationId: listOrders
      parameters:
        - name: api_version
          in: query
          required: false
          schema:
            type: string
            enum: ['1', '2']
            default: '2'
          description: API version
      responses:
        '200':
          description: List of orders
          content:
            application/json:
              schema:
                type: object
                properties:
                  data:
                    type: array
                    items:
                      $ref: '#/components/schemas/Order'

Comparison

Criteria URL Path Header Query Param
Discoverability High (visible in URL) Low (hidden in headers) Medium
Cache-friendly Yes (different URL per version) Requires Vary header Yes (different URL per version)
Browser testable Yes No (needs tools) Yes
URL cleanliness Adds /v2 prefix Clean URLs Adds ?api_version=2
Spec complexity Separate spec per version Single spec, conditional logic Single spec, conditional logic
Used by Stripe, GitHub, Twilio Azure, GitHub (Accept header) Google, Amazon

How Delimit Enforces Version Policies

Delimit supports path-scoped policy rules, so you can enforce different policies per API version. This is especially powerful with URL path versioning.

# .delimit/policies.yml — Version-scoped rules
rules:
  # Strict policy for stable v2 paths
  - path: /v2/**
    breaking_changes: error
    deprecation_required: true

  # Relaxed policy for beta v3 paths
  - path: /v3/**
    breaking_changes: warn
    deprecation_required: false

  # Block removal of versioned endpoints
  - change: endpoint-removed
    severity: error
    message: "Endpoints must be deprecated before removal"

Delimit detects version-related breaking changes:

Change Detection Severity
Remove /v1/orders endpoint endpoint-removed Breaking
Remove version enum value from header enum-value-removed Breaking
Change version parameter from optional to required param-required-changed Breaking
Add new /v3/orders endpoint endpoint-added Non-breaking
Add deprecation notice to /v1 endpoints deprecated-added Non-breaking

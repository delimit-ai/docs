OpenAPI Filtering and Search Pattern

Query parameter patterns for filtering, sorting, and full-text search. Define them explicitly in your OpenAPI spec so consumers know exactly what is supported and Delimit can protect the contract.

1. Field-Level Filtering

Individual query parameters for each filterable field. The most explicit and self-documenting approach.

# OpenAPI 3.0 — Field-Level Filters
paths:
  /orders:
    get:
      summary: List orders with filters
      parameters:
        - name: status
          in: query
          schema:
            type: string
            enum: [pending, processing, shipped, delivered, cancelled]
          description: Filter by order status
        - name: customer_id
          in: query
          schema:
            type: string
            format: uuid
          description: Filter by customer
        - name: created_after
          in: query
          schema:
            type: string
            format: date-time
          description: Orders created after this timestamp
        - name: created_before
          in: query
          schema:
            type: string
            format: date-time
          description: Orders created before this timestamp
        - name: min_total
          in: query
          schema:
            type: number
            format: float
            minimum: 0
          description: Minimum order total
        - name: max_total
          in: query
          schema:
            type: number
            format: float
          description: Maximum order total
      responses:
        '200':
          description: Filtered list of orders
          content:
            application/json:
              schema:
                type: object
                properties:
                  data:
                    type: array
                    items:
                      $ref: '#/components/schemas/Order'
                  meta:
                    $ref: '#/components/schemas/PaginationMeta'

2. Sorting

Use a sort parameter with defined allowed values. Support ascending and descending order with a prefix convention.

# OpenAPI 3.0 — Sorting
parameters:
  - name: sort
    in: query
    schema:
      type: string
      enum:
        - created_at
        - -created_at
        - total
        - -total
        - status
        - -status
      default: -created_at
    description: |
      Sort field. Prefix with - for descending order.
      Examples: "created_at" (ascending), "-created_at" (descending)
  - name: sort_by
    in: query
    schema:
      type: string
      enum: [created_at, total, status, updated_at]
    description: Field to sort by (alternative style)
  - name: sort_order
    in: query
    schema:
      type: string
      enum: [asc, desc]
      default: desc
    description: Sort direction

3. Full-Text Search

A q or search parameter for free-text queries. Keep it simple and document what fields are searched.

# OpenAPI 3.0 — Full-Text Search
paths:
  /products:
    get:
      summary: Search products
      parameters:
        - name: q
          in: query
          schema:
            type: string
            minLength: 2
            maxLength: 200
          description: |
            Full-text search query. Searches across name,
            description, and SKU fields.
          example: "wireless headphones"
        - name: category
          in: query
          schema:
            type: string
            enum: [electronics, clothing, books, home]
          description: Filter results by category
        - name: in_stock
          in: query
          schema:
            type: boolean
          description: Only return items currently in stock
      responses:
        '200':
          description: Search results
          content:
            application/json:
              schema:
                type: object
                properties:
                  data:
                    type: array
                    items:
                      $ref: '#/components/schemas/Product'
                  meta:
                    type: object
                    properties:
                      total_results:
                        type: integer
                      query:
                        type: string
                      took_ms:
                        type: integer
                        description: Search execution time in milliseconds

4. Compound Filter Object

For complex filtering, use a structured filter parameter with JSON encoding or bracket syntax.

# OpenAPI 3.0 — Structured Filter Parameter
paths:
  /events:
    get:
      summary: List events with compound filters
      parameters:
        - name: filter[type]
          in: query
          schema:
            type: string
            enum: [concert, conference, meetup, workshop]
        - name: filter[city]
          in: query
          schema:
            type: string
        - name: filter[date_gte]
          in: query
          schema:
            type: string
            format: date
          description: Events on or after this date
        - name: filter[date_lte]
          in: query
          schema:
            type: string
            format: date
          description: Events on or before this date
        - name: filter[tags]
          in: query
          schema:
            type: array
            items:
              type: string
          style: form
          explode: true
          description: Filter by tags (multiple allowed)
      responses:
        '200':
          description: Filtered events

5. Field Selection (Sparse Fieldsets)

Let clients request only the fields they need to reduce payload size.

# OpenAPI 3.0 — Field Selection
parameters:
  - name: fields
    in: query
    schema:
      type: string
    description: |
      Comma-separated list of fields to include in response.
      Example: "id,name,email,created_at"
    example: "id,name,email"

Breaking Changes Delimit Catches

Filter parameters are part of your API contract. Consumers build query logic around the available filters. Delimit detects:

Change Detection Severity
Remove status filter parameter param-removed Breaking
Remove shipped from status enum enum-value-removed Breaking
Change sort type from string to array param-type-changed Breaking
Make q parameter required param-required-changed Breaking
Decrease q maxLength from 200 to 100 max-length-decreased Breaking
Increase q minLength from 2 to 5 min-length-increased Breaking
Add new optional region filter optional-param-added Non-breaking
Add new enum value returned to status enum-value-added Non-breaking

OpenAPI Pagination Pattern

Cursor, offset, and keyset pagination in OpenAPI. Choose the right strategy, define it correctly in your spec, and let Delimit catch breaking changes automatically.

1. Offset-Based Pagination

The simplest approach. Clients pass offset and limit parameters. Common in SQL-backed APIs where OFFSET maps directly to a query clause.

# OpenAPI 3.0 — Offset Pagination
paths:
  /items:
    get:
      summary: List items with offset pagination
      parameters:
        - name: offset
          in: query
          schema:
            type: integer
            minimum: 0
            default: 0
          description: Number of items to skip
        - name: limit
          in: query
          schema:
            type: integer
            minimum: 1
            maximum: 100
            default: 20
          description: Maximum number of items to return
      responses:
        '200':
          description: Paginated list of items
          content:
            application/json:
              schema:
                type: object
                required:
                  - data
                  - total
                  - offset
                  - limit
                properties:
                  data:
                    type: array
                    items:
                      $ref: '#/components/schemas/Item'
                  total:
                    type: integer
                    description: Total number of items
                  offset:
                    type: integer
                  limit:
                    type: integer

2. Cursor-Based Pagination

Clients pass an opaque cursor token to fetch the next page. Scales well for large datasets and real-time feeds where rows may be inserted between requests.

# OpenAPI 3.0 — Cursor Pagination
paths:
  /events:
    get:
      summary: List events with cursor pagination
      parameters:
        - name: cursor
          in: query
          schema:
            type: string
          description: Opaque cursor from previous response
        - name: limit
          in: query
          schema:
            type: integer
            minimum: 1
            maximum: 100
            default: 20
      responses:
        '200':
          description: Paginated list of events
          content:
            application/json:
              schema:
                type: object
                required:
                  - data
                  - next_cursor
                  - has_more
                properties:
                  data:
                    type: array
                    items:
                      $ref: '#/components/schemas/Event'
                  next_cursor:
                    type: string
                    nullable: true
                    description: Cursor for the next page, null if no more pages
                  has_more:
                    type: boolean
                    description: Whether more pages exist

3. Keyset Pagination

A variant of cursor pagination that uses a known sort key (like created_at or id) instead of an opaque token. Transparent and debuggable.

# OpenAPI 3.0 — Keyset Pagination
paths:
  /orders:
    get:
      summary: List orders with keyset pagination
      parameters:
        - name: after_id
          in: query
          schema:
            type: string
            format: uuid
          description: Return orders created after this ID
        - name: limit
          in: query
          schema:
            type: integer
            minimum: 1
            maximum: 100
            default: 20
        - name: sort
          in: query
          schema:
            type: string
            enum: [created_at, updated_at]
            default: created_at
      responses:
        '200':
          description: Paginated list of orders
          content:
            application/json:
              schema:
                type: object
                required:
                  - data
                  - has_more
                properties:
                  data:
                    type: array
                    items:
                      $ref: '#/components/schemas/Order'
                  has_more:
                    type: boolean
                  next_after_id:
                    type: string
                    format: uuid
                    nullable: true

Comparison

Criteria Offset Cursor Keyset
Simplicity Simple, familiar Moderate Moderate
Performance at scale Degrades with large offsets Consistent O(1) Consistent O(1)
Real-time stability Skips or duplicates on inserts Stable Stable
Random page access Yes (jump to any page) No (sequential only) No (sequential only)
Total count available Yes (via total) Usually no Usually no
Debuggability High Low (opaque token) High (visible key)
Best for Small datasets, admin UIs Feeds, event streams Large ordered datasets

Breaking Changes Delimit Catches

Pagination fields are part of your API contract. Delimit detects these breaking changes automatically:

Change Detection Severity
Remove cursor or next_cursor field field-removed Breaking
Remove offset or limit parameter param-removed Breaking
Change limit max from 100 to 50 max-length-decreased Breaking
Make cursor parameter required param-required-changed Breaking
Remove has_more from response field-removed Breaking
Add new optional parameter page_token optional-param-added Non-breaking

OpenAPI Authentication Patterns

API key, OAuth2, and JWT bearer security schemes in OpenAPI. Define authentication once, apply globally or per-operation, and let Delimit catch auth breaking changes.

1. API Key Authentication

The simplest auth scheme. The key can be sent in a header, query parameter, or cookie. Header-based is most common.

# OpenAPI 3.0 — API Key in Header
components:
  securitySchemes:
    ApiKeyAuth:
      type: apiKey
      in: header
      name: X-API-Key
      description: API key provided in your dashboard

# Apply globally to all endpoints
security:
  - ApiKeyAuth: []

# Or apply per-operation
paths:
  /public/health:
    get:
      summary: Health check (no auth required)
      security: []  # Override: no auth
      responses:
        '200':
          description: OK
  /users:
    get:
      summary: List users (requires API key)
      # Inherits global ApiKeyAuth
      responses:
        '200':
          description: List of users

2. OAuth2 Authentication

Full OAuth2 flow definitions including authorization code, client credentials, and implicit grants. Define scopes to control granular access.

# OpenAPI 3.0 — OAuth2 with Scopes
components:
  securitySchemes:
    OAuth2:
      type: oauth2
      flows:
        authorizationCode:
          authorizationUrl: https://auth.example.com/authorize
          tokenUrl: https://auth.example.com/token
          refreshUrl: https://auth.example.com/token
          scopes:
            read:users: Read user profiles
            write:users: Create and update users
            read:orders: Read order data
            write:orders: Create and manage orders
            admin: Full administrative access
        clientCredentials:
          tokenUrl: https://auth.example.com/token
          scopes:
            read:users: Read user profiles
            read:orders: Read order data

# Apply with required scopes
paths:
  /users:
    get:
      summary: List users
      security:
        - OAuth2: [read:users]
      responses:
        '200':
          description: List of users
    post:
      summary: Create user
      security:
        - OAuth2: [write:users]
      responses:
        '201':
          description: User created
  /admin/settings:
    put:
      summary: Update system settings
      security:
        - OAuth2: [admin]
      responses:
        '200':
          description: Settings updated

3. JWT Bearer Authentication

Uses the HTTP bearer scheme with JWTs. Common for service-to-service communication and SPAs.

# OpenAPI 3.0 — JWT Bearer
components:
  securitySchemes:
    BearerAuth:
      type: http
      scheme: bearer
      bearerFormat: JWT
      description: |
        JWT token obtained from /auth/token endpoint.
        Include in Authorization header: Bearer <token>

# Global application
security:
  - BearerAuth: []

paths:
  /auth/token:
    post:
      summary: Obtain JWT token
      security: []  # No auth required to get a token
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              required:
                - email
                - password
              properties:
                email:
                  type: string
                  format: email
                password:
                  type: string
                  format: password
      responses:
        '200':
          description: JWT token
          content:
            application/json:
              schema:
                type: object
                properties:
                  access_token:
                    type: string
                  token_type:
                    type: string
                    example: bearer
                  expires_in:
                    type: integer
                    example: 3600
                  refresh_token:
                    type: string

4. Multiple Auth Schemes (AND / OR)

OpenAPI supports combining auth schemes. Items in the same array element are AND; separate array elements are OR.

# AND: Require both API key AND OAuth2
security:
  - ApiKeyAuth: []
    OAuth2: [read:users]

# OR: Accept either API key OR Bearer token
security:
  - ApiKeyAuth: []
  - BearerAuth: []

Breaking Changes Delimit Catches

Authentication changes are among the most disruptive breaking changes. Every consumer must update their auth configuration. Delimit detects:

Change Detection Severity
Remove a security scheme entirely security-removed Breaking
Remove an OAuth2 scope (read:users) security-scope-removed Breaking
Add auth requirement to a previously public endpoint security-added Breaking
Change API key header name from X-API-Key to Authorization security-removed + security-added Breaking
Add a new optional security scheme security-added Context-dependent
Remove auth from an endpoint (make it public) security-removed Context-dependent

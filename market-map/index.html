Where Delimit Fits

Delimit is the CI enforcement layer for API compatibility

Delimit sits between your OpenAPI spec and your merge button. It catches breaking changes before they reach production, enforces team policies on what changes are allowed, and gives every PR a deterministic pass/fail verdict with migration guidance.

What Delimit replaces

  • Manual spec review for breaking changes
  • Undocumented "check the diff" processes
  • Post-deployment breaking change discovery
  • Ad-hoc semver bump decisions

What Delimit does not replace

Tool category Example Why Delimit does not replace it
Linter Spectral, Redocly Linters check spec quality against style rules. Delimit checks spec compatibility between versions. Run both.
Documentation Redoc, Swagger UI Delimit detects changes, not renders docs. Use a doc renderer to publish your API reference.
API testing Postman, Bruno Delimit works at the spec level. API testers validate runtime behavior against live endpoints.
Gateway Kong, Tyk, Apigee Gateways manage production traffic. Delimit operates pre-merge in CI. Different lifecycle stages.
Design editor Stoplight Studio Design tools help you author specs. Delimit validates changes to specs you have already written.

The recommended stack

A complete API governance pipeline uses tools from multiple categories. Here is one recommended combination:

  1. Design — Author your OpenAPI spec in Stoplight Studio or your editor of choice.
  2. Lint — Run Spectral or Redocly CLI to enforce spec quality and naming conventions.
  3. Diff & enforce — Run delimit lint in CI to catch breaking changes, enforce policies, and classify the semver impact.
  4. Document — Render your spec with Redoc or Swagger UI for consumer-facing reference docs.
  5. Test — Use Postman or Bruno to validate that the live API matches the spec.
  6. Manage — Route production traffic through Kong, Tyk, or your gateway of choice.

Get Started

npx delimit-cli setup

One command adds Delimit to any repo with an OpenAPI spec. No account, no API key, no configuration file required.

Docs / Templates

AI Assistant Templates

Drop-in CLAUDE.md configuration templates with Delimit governance hooks for 10 popular API frameworks. Each template includes framework-specific instructions for route locations, spec generation, testing commands, and common breaking change patterns.

How to Use

  1. Download the template for your framework (or copy its contents).
  2. Save it as CLAUDE.md in your project root.
  3. Customize the framework-specific sections for your project structure.
  4. AI assistants (Claude Code, Codex, Gemini CLI) will follow the governance rules automatically.

What Each Template Includes

Section Purpose
Delimit Governance Session start hooks, post-edit lint calls, pre-commit diagnostics
Tool Routing MCP tool requirements for secrets, tasks, and memory (audit trail enforcement)
Project Structure Standard directory layout for the framework
Route Locations Where API routes are defined and how they map to paths
Spec Generation How to generate/export the OpenAPI spec for that framework
Testing Commands Framework-specific test runner commands with coverage
Breaking Change Patterns Common code changes that produce breaking API changes
Pre-Commit Hook Ready-to-use hook configuration for CI integration

Customization

Each template is a starting point. After downloading, you should:

  • Update file paths to match your project structure
  • Add project-specific coding conventions and style rules
  • Configure the OpenAPI spec export for your setup
  • Add any additional Delimit MCP tools relevant to your workflow

Works With

These templates are designed for CLAUDE.md but the governance patterns work with any AI coding assistant that reads project-level configuration:

  • Claude Code — reads CLAUDE.md natively
  • OpenAI Codex — reads AGENTS.md (rename the file)
  • Gemini CLI — reads GEMINI.md (rename the file)
  • Cursor / Windsurf — reads .cursorrules / .windsurfrules

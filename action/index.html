GitHub Action

Add API governance to your CI pipeline with the Delimit GitHub Action.

Installation

Available on the GitHub Marketplace.

Basic Setup

Create .github/workflows/api-governance.yml:

name: API Governance
on:
  pull_request:
    paths: ["openapi.yaml"]

jobs:
  delimit:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: delimit-ai/delimit-action@v1
        with:
          spec: openapi.yaml

Inputs

InputDescriptionDefault
specPath to OpenAPI spec. Delimit auto-fetches the base branch version.auto-detect
old_specPath to old/base spec (advanced mode).
new_specPath to new/changed spec (advanced mode).
modeadvisory (comments only) or enforce (fail CI).advisory
github_tokenToken for PR comments.${{ github.token }}
policy_filePath to custom policy file.

Outputs

OutputDescription
breaking_changes_detectedtrue or false
violations_countNumber of policy violations
semver_bumpmajor, minor, patch, or none
next_versionComputed next version string
reportFull JSON report of all changes

Advisory vs Enforce Mode

Advisory mode (default): Delimit posts a PR comment with the diff report but does not block the merge. Good for initial adoption.

Enforce mode: Delimit blocks the merge if breaking changes are detected. Use this once your team is comfortable with the governance workflow.

- uses: delimit-ai/delimit-action@v1
  with:
    spec: openapi.yaml
    mode: enforce

Advanced: Two-Spec Mode

For monorepos or custom setups where you manage both spec files:

- uses: delimit-ai/delimit-action@v1
  with:
    old_spec: specs/api-v1.yaml
    new_spec: specs/api-v2.yaml
    mode: enforce
    policy_file: .delimit/policies.yml

Auto-Detection

If no spec input is provided, Delimit searches for common spec file names: openapi.yaml, openapi.yml, openapi.json, swagger.yaml, etc. in the root and common subdirectories (api/, docs/, spec/).

PR Comment

Delimit automatically posts (and updates) a PR comment showing:

  • Breaking changes with severity levels
  • Semver recommendation
  • Migration guide for each breaking change
  • Additive (non-breaking) changes

Change Types » endpoint_added

endpoint_added

Non-Breaking Severity: low

A new endpoint has been added to the API. This is purely additive and does not affect existing consumers.

Why Is This Non-Breaking?

This is a non-breaking change. New endpoints do not affect existing consumers. They can start using the new endpoint at their convenience.

Example

Before

paths:
  /users:
    get:
      summary: List users

After

paths:
  /users:
    get:
      summary: List users
  /teams:
    get:
      summary: List teams

How Delimit Detects It

Delimit compares the paths keys between specs. Any path present in the new spec but absent in the old spec is flagged as endpoint_added.

Migration Guide

No migration needed. Document the new endpoint and notify consumers of the new capability. Update SDKs and client libraries to expose the new functionality.

Related Change Types

  • endpoint_removed
  • method_added

Detect This Change

npx delimit-cli lint --old old-spec.yaml --new new-spec.yaml

Or add the GitHub Action to catch this automatically on every pull request.

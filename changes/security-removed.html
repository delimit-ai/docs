Change Types » security_removed

security_removed

Breaking Severity: high

A security scheme has been removed from the spec or from an operation. Consumers configured to authenticate with this scheme will need to change their authentication approach.

Why Is This Breaking?

Removing a security scheme breaks consumers that are configured to use it. Their authentication tokens or credentials become invalid for the API, and they must reconfigure their authentication.

Example

Before

components:
  securitySchemes:
    ApiKeyAuth:
      type: apiKey
      in: header
      name: X-API-Key
    BearerAuth:
      type: http
      scheme: bearer

After

components:
  securitySchemes:
    BearerAuth:
      type: http
      scheme: bearer
    # ApiKeyAuth removed

How Delimit Detects It

Delimit compares both global security schemes (under components/securitySchemes) and per-operation security requirements. Schemes present in the old spec but absent in the new spec are flagged.

Migration Guide

1. Announce the deprecation of the old auth scheme with a timeline. 2. Support both authentication methods during a transition period. 3. Provide migration documentation showing the new authentication flow. 4. Invalidate old credentials only after the transition period ends.

Detect This Change

npx delimit-cli lint --old old-spec.yaml --new new-spec.yaml

Or add the GitHub Action to catch this automatically on every pull request.

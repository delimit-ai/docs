Change Types » param_type_changed

param_type_changed

Breaking Severity: high

The data type of a parameter has changed (e.g., query parameter changed from string to integer). Consumers sending the old type will fail validation.

Why Is This Breaking?

Consumers construct parameter values based on the documented type. Changing a query parameter from string to integer means existing string-based requests will fail server-side validation.

Example

Before

parameters:
  - name: user_id
    in: query
    schema:
      type: string

After

parameters:
  - name: user_id
    in: query
    schema:
      type: integer

How Delimit Detects It

Delimit compares the schema.type of parameters that exist in both old and new specs. A type mismatch triggers both param_type_changed and the general type_changed change.

Migration Guide

1. Add a new parameter with the new type and deprecate the old one. 2. Accept both types server-side during a transition period. 3. If string-to-number, consider continuing to accept quoted numbers. 4. Version the endpoint if the type change is fundamental.

Detect This Change

npx delimit-cli lint --old old-spec.yaml --new new-spec.yaml

Or add the GitHub Action to catch this automatically on every pull request.

Change Types » enum_value_added

enum_value_added

Non-Breaking Severity: low

A new value has been added to an enum list. Existing values continue to work. Consumers should handle unknown enum values gracefully.

Why Is This Non-Breaking?

This is a non-breaking change in principle, though consumers with strict enum validation may need updates. Well-designed clients should handle unknown enum values.

Example

Before

properties:
  status:
    type: string
    enum: [active, inactive]

After

properties:
  status:
    type: string
    enum: [active, inactive, pending]

How Delimit Detects It

Delimit compares enum arrays. Values present in the new spec but absent in the old spec are flagged as enum_value_added.

Migration Guide

No migration strictly needed. Consumers should ensure they handle unknown enum values gracefully. Update any client-side enum types or validation to include the new value.

Related Change Types

  • enum_value_removed

Detect This Change

npx delimit-cli lint --old old-spec.yaml --new new-spec.yaml

Or add the GitHub Action to catch this automatically on every pull request.

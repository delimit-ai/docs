Change Types » param_required_changed

param_required_changed

Breaking Severity: high

A parameter has changed from optional to required. Existing clients that do not send this parameter will start receiving validation errors.

Why Is This Breaking?

Consumers that previously omitted this optional parameter will now receive 400 errors because the server requires it. This is functionally identical to adding a new required parameter for affected consumers.

Example

Before

parameters:
  - name: api_version
    in: header
    required: false
    schema:
      type: string

After

parameters:
  - name: api_version
    in: header
    required: true
    schema:
      type: string

How Delimit Detects It

Delimit compares the required flag on parameters that exist in both old and new specs. A change from false (or absent) to true is flagged.

Migration Guide

1. Instead of making it required, add a default value so existing requests work. 2. If it must be required, provide a migration period where the server supplies a default. 3. Communicate the change well in advance. 4. Consider using API versioning to introduce the requirement.

npx delimit-cli lint --old old-spec.yaml --new new-spec.yaml

Or add the GitHub Action to catch this automatically on every pull request.

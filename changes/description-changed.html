Change Types » description_changed

description_changed

Non-Breaking Severity: low

A description field has been modified. This is a documentation-only change with no functional impact on consumers.

Why Is This Non-Breaking?

This is a non-breaking change. Description changes are purely cosmetic and do not affect API behavior, request/response formats, or validation.

Example

Before

paths:
  /users:
    get:
      description: Get all users

After

paths:
  /users:
    get:
      description: Retrieve a paginated list of all users in the organization

How Delimit Detects It

Delimit compares description fields on operations, parameters, schemas, and properties. Any text change is flagged as description_changed.

Migration Guide

No migration needed. Updated descriptions improve documentation quality. Regenerate SDK documentation if auto-generated.

Detect This Change

npx delimit-cli lint --old old-spec.yaml --new new-spec.yaml

Or add the GitHub Action to catch this automatically on every pull request.

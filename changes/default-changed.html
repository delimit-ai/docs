Change Types » default_changed

default_changed

Non-Breaking Severity: low

The default value of a parameter or field has changed. Consumers relying on the old default behavior will see different results.

Why Is This Non-Breaking?

Classified as non-breaking because consumers explicitly setting the value are unaffected. However, consumers relying on the default (by omitting the parameter) will experience behavior changes. Review carefully.

Example

Before

parameters:
  - name: limit
    in: query
    schema:
      type: integer
      default: 20

After

parameters:
  - name: limit
    in: query
    schema:
      type: integer
      default: 50

How Delimit Detects It

Delimit compares default values on parameters and schema properties. Any change in the default value is flagged as default_changed.

Migration Guide

1. Communicate the default value change to consumers. 2. Consumers relying on the default should explicitly set the old value if they need the previous behavior. 3. Update documentation to reflect the new default. 4. Consider whether this warrants a MINOR version bump in your API.

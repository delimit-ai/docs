Change Types » endpoint_removed

endpoint_removed

Breaking Severity: high

An entire API endpoint (URL path) has been removed from the specification. All consumers calling this endpoint will receive 404 errors.

Why Is This Breaking?

Any client that sends requests to the removed endpoint will immediately fail. There is no graceful fallback -- the server will return a 404 Not Found or similar error, breaking all integrations that depend on it.

Example

Before

paths:
  /users:
    get:
      summary: List users
  /users/{id}:
    get:
      summary: Get user by ID

After

paths:
  /users:
    get:
      summary: List users
  # /users/{id} has been removed

How Delimit Detects It

Delimit compares the paths keys between the old and new OpenAPI specs. Any path present in the old spec but absent in the new spec is flagged as endpoint_removed.

Migration Guide

1. Announce the deprecation with a timeline (e.g., 90 days). 2. Add a deprecated: true flag to the endpoint first. 3. Return 410 Gone with a body indicating the replacement. 4. Provide a replacement endpoint or migration guide. 5. Only remove the endpoint after the deprecation period.

Detect This Change

npx delimit-cli lint --old old-spec.yaml --new new-spec.yaml

Or add the GitHub Action to catch this automatically on every pull request.

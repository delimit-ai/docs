Change Types » deprecated_added

deprecated_added

Non-Breaking Severity: low

An operation or field has been marked as deprecated: true. This signals intent to remove but does not break existing consumers.

Why Is This Non-Breaking?

This is a non-breaking change. Deprecation is a warning, not a removal. The deprecated element continues to function. It signals that consumers should plan to migrate.

Example

Before

paths:
  /users/search:
    get:
      summary: Search users

After

paths:
  /users/search:
    get:
      summary: Search users
      deprecated: true

How Delimit Detects It

Delimit compares the deprecated flag on operations and schema properties. A change from false (or absent) to true is flagged.

Migration Guide

No immediate migration needed. Plan to migrate away from the deprecated element before it is removed. Check the API changelog for the planned removal date.

Detect This Change

npx delimit-cli lint --old old-spec.yaml --new new-spec.yaml

Or add the GitHub Action to catch this automatically on every pull request.

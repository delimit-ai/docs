Change Types » optional_param_added

optional_param_added

Non-Breaking Severity: low

A new optional parameter has been added to an operation. Existing clients will continue to work without changes.

Why Is This Non-Breaking?

This is a non-breaking change. Optional parameters have defaults or can be omitted. Existing consumers are unaffected and can adopt the parameter at their own pace.

Example

Before

parameters:
  - name: limit
    in: query
    required: false
    schema:
      type: integer

After

parameters:
  - name: limit
    in: query
    required: false
    schema:
      type: integer
  - name: cursor
    in: query
    required: false
    schema:
      type: string

How Delimit Detects It

Delimit checks new parameters for their required flag. Parameters with required: false (or no required flag, which defaults to false) are classified as optional additions.

Migration Guide

No migration needed. Document the new parameter with its default behavior. Update SDKs to expose the new option.

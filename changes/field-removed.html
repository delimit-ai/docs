Change Types » field_removed

field_removed

Breaking Severity: high

A field has been removed from a schema (request body, response, or component). Consumers that read or write this field will break.

Why Is This Breaking?

Removing a field from a response breaks consumers that read it. Removing a field from a request body means consumers sending it will either get errors or have data silently dropped.

Example

Before

components:
  schemas:
    User:
      type: object
      properties:
        id:
          type: string
        name:
          type: string
        legacy_role:
          type: string

After

components:
  schemas:
    User:
      type: object
      properties:
        id:
          type: string
        name:
          type: string
        # legacy_role removed

How Delimit Detects It

Delimit recursively compares object properties in schemas. A property present in the old spec's properties but absent in the new spec is flagged, especially if it was in the required array.

Migration Guide

1. Mark the field as deprecated: true before removing it. 2. Continue returning the field with a null or default value. 3. For response fields, add a sunset header indicating when the field will be removed. 4. Provide a replacement field if applicable.

Related Change Types

  • optional_field_added

Detect This Change

npx delimit-cli lint --old old-spec.yaml --new new-spec.yaml

Or add the GitHub Action to catch this automatically on every pull request.

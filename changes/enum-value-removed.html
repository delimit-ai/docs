Change Types » enum_value_removed

enum_value_removed

Breaking Severity: high

A value has been removed from an enum list. Consumers sending the removed value will receive validation errors; consumers parsing it will encounter unexpected behavior.

Why Is This Breaking?

Consumers may be sending the removed enum value in requests (causing 400 errors) or expecting to receive it in responses (causing parsing failures). Either direction is a breaking change.

Example

Before

properties:
  status:
    type: string
    enum: [active, inactive, pending, archived]

After

properties:
  status:
    type: string
    enum: [active, inactive, pending]
    # "archived" removed

How Delimit Detects It

Delimit compares enum arrays on parameters and schema properties. Values present in the old enum set but absent in the new set are flagged.

Migration Guide

1. Keep the old enum value but mark it as deprecated in the description. 2. Map the old value to a new value server-side during a transition period. 3. Return a warning header when the deprecated value is used. 4. Only remove after confirming no consumers send or expect this value.

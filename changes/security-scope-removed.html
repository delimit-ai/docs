Change Types » security_scope_removed

security_scope_removed

Breaking Severity: high

An OAuth scope has been removed from a security requirement. Consumers whose tokens include this scope may experience authorization changes.

Why Is This Breaking?

Removing an OAuth scope can change authorization behavior. Consumers may have tokens that were specifically granted this scope, and removing it from the spec may indicate the server no longer recognizes it.

Example

Before

security:
  - OAuth2:
    - read:users
    - write:users
    - admin:users

After

security:
  - OAuth2:
    - read:users
    - write:users
    # admin:users scope removed

How Delimit Detects It

Delimit compares the scope arrays within security requirements for each scheme. Scopes present in the old spec but absent in the new spec are flagged.

Migration Guide

1. Map the old scope to an equivalent new scope server-side. 2. Continue accepting the old scope during a transition period. 3. Notify consumers to request updated tokens. 4. Update OAuth consent screens and documentation.

Detect This Change

npx delimit-cli lint --old old-spec.yaml --new new-spec.yaml

Or add the GitHub Action to catch this automatically on every pull request.

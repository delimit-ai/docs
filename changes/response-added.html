Change Types » response_added

response_added

Non-Breaking Severity: low

A new response status code has been added to an operation. This is additive and does not affect existing consumers.

Why Is This Non-Breaking?

This is a non-breaking change. Properly implemented clients should handle unexpected status codes gracefully. The new response provides additional information about the API's behavior.

Example

Before

responses:
  "200":
    description: Success

After

responses:
  "200":
    description: Success
  "429":
    description: Rate limited

How Delimit Detects It

Delimit compares response status codes between old and new specs. New status codes are flagged as response_added.

Migration Guide

No migration needed. Update client error handling to take advantage of the new response code for better error messages and retry logic.

Related Change Types

  • response_removed
  • response_type_changed

Detect This Change

npx delimit-cli lint --old old-spec.yaml --new new-spec.yaml

Or add the GitHub Action to catch this automatically on every pull request.

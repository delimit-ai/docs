Change Types » min_length_increased

min_length_increased

Breaking Severity: high

A minLength or minItems constraint has been increased or newly added. Values that were previously valid may now be rejected.

Why Is This Breaking?

Consumers sending values that met the old minimum but fall below the new minimum will have their requests rejected. This affects any consumer providing short values or empty collections.

Example

Before

properties:
  username:
    type: string
    minLength: 1

After

properties:
  username:
    type: string
    minLength: 3

How Delimit Detects It

Delimit compares minLength and minItems constraints. An increase in value, or adding a min constraint where none existed (greater than 0), is flagged.

Migration Guide

1. Validate existing data against the new constraint before deploying. 2. Pad or migrate values that fall below the new minimum. 3. Apply the new constraint only to new values, grandfathering existing ones. 4. Provide clear error messages explaining the new minimum requirement.

npx delimit-cli lint --old old-spec.yaml --new new-spec.yaml

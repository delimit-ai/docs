Change Types » method_removed

method_removed

Breaking Severity: high

An HTTP method (GET, POST, PUT, DELETE, PATCH, etc.) has been removed from an existing endpoint. Consumers using that method will receive 405 Method Not Allowed errors.

Why Is This Breaking?

Clients explicitly choose HTTP methods for their requests. Removing a method means any consumer relying on it will fail with a 405 error, even though the endpoint path still exists.

Example

Before

paths:
  /users/{id}:
    get:
      summary: Get user
    delete:
      summary: Delete user

After

paths:
  /users/{id}:
    get:
      summary: Get user
    # DELETE method removed

How Delimit Detects It

Delimit iterates over each shared path and compares the HTTP methods (get, post, put, delete, patch, head, options). Any method present in the old spec but absent in the new spec triggers this change.

Migration Guide

1. Mark the method as deprecated: true before removal. 2. Return 405 Method Not Allowed with an Allow header listing valid methods. 3. Document the alternative approach (e.g., soft-delete via PATCH). 4. Give consumers time to migrate before removing the method.

Detect This Change

npx delimit-cli lint --old old-spec.yaml --new new-spec.yaml

Or add the GitHub Action to catch this automatically on every pull request.

Change Types » optional_field_added

optional_field_added

Non-Breaking Severity: low

A new optional field has been added to a schema. Existing clients will continue to work; the new field can be adopted at their convenience.

Why Is This Non-Breaking?

This is a non-breaking change. Optional fields can be omitted in requests and may or may not appear in responses. Well-behaved clients should ignore unknown fields.

Example

Before

components:
  schemas:
    User:
      type: object
      properties:
        id:
          type: string
        name:
          type: string

After

components:
  schemas:
    User:
      type: object
      properties:
        id:
          type: string
        name:
          type: string
        avatar_url:
          type: string

How Delimit Detects It

Delimit checks new properties in schemas. Properties that are not in the required array are classified as optional field additions.

Migration Guide

No migration needed. Update SDKs to include the new field. Consumers can start using it when ready.

Or add the GitHub Action to catch this automatically on every pull request.

Change Types » param_removed

param_removed

Breaking Severity: high

An existing parameter has been removed from an operation. Clients sending this parameter may receive errors or experience silent behavior changes.

Why Is This Breaking?

Consumers that include the removed parameter may get validation errors (strict servers) or have their parameter silently ignored (lenient servers). Either way, the intended behavior changes.

Example

Before

parameters:
  - name: sort_by
    in: query
    schema:
      type: string
  - name: include_deleted
    in: query
    schema:
      type: boolean

After

parameters:
  - name: sort_by
    in: query
    schema:
      type: string
  # include_deleted parameter removed

How Delimit Detects It

Delimit identifies parameters by their name and in location (query, header, path, cookie). A parameter present in the old spec but absent in the new spec is flagged.

Migration Guide

1. Deprecate the parameter first by adding deprecated: true. 2. Continue accepting the parameter but ignore it server-side. 3. Log a warning when the deprecated parameter is received. 4. Remove after a deprecation period.

Detect This Change

npx delimit-cli lint --old old-spec.yaml --new new-spec.yaml

Or add the GitHub Action to catch this automatically on every pull request.
